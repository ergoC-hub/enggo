# EngGo — API 무료 운영 설정 가이드

구글 경고의 원인이었던 **키 노출 문제**를 고치고, 무료 등급으로 안전하게 운영하기 위한 설정 순서입니다.

## 왜 경고를 받았나?

1. **이전 버전 코드에 API 키가 그대로 적혀 있었습니다** (`enggo-pwa/index.html`).
   GitHub에 올라가면 구글/Anthropic이 자동 스캔해서 경고를 보내고 키를 정지시킵니다.
2. **현재 버전도 사실상 키가 공개 상태였습니다.**
   Supabase `app_config` 테이블에 Gemini 키를 넣고 앱이 내려받는 구조였는데,
   이 테이블은 누구나(anon 키만 있으면) 읽을 수 있어서 키를 그대로 가져갈 수 있습니다.

> 핵심 원칙: **브라우저(클라이언트)에 도달하는 키는 전부 공개된 것**입니다.
> 키는 서버에만 두고, 앱은 서버를 거쳐서 AI를 호출해야 합니다.

## 새 구조

```
[앱 (GitHub Pages)] ──로그인 토큰──▶ [Supabase Edge Function: gemini-proxy] ──키──▶ [Gemini API]
                                          │
                                          └─ 1인당 하루 40회 제한 (무료 한도 보호)
```

- Gemini 키는 Supabase **서버 비밀값(secret)** 에만 저장 → 앱 코드/DB에 키 없음
- **로그인한 사용자만** AI 호출 가능
- 1인당 일일 한도 + 호출 간격 제한으로 무료 등급(RPM/RPD) 보호
- 모델은 `gemini-2.5-flash-lite` (무료 일일 한도가 가장 넉넉함)

## 설정 순서

### 1단계 — 유출된 키 폐기 & 새 키 발급
1. https://aistudio.google.com/app/apikey 접속
2. 경고받은 기존 키 **삭제** (이미 삭제했다면 통과)
3. **새 키 발급** — 이 키는 절대 코드/DB에 넣지 않습니다

### 2단계 — Supabase SQL 실행
1. Supabase 대시보드 → **SQL Editor**
2. `supabase/setup.sql` 내용 전체를 붙여넣고 실행
   - AI 사용량 테이블 + 한도 함수 생성
   - 본인 데이터만 쓸 수 있게 RLS 정책 적용 (XP/랭킹 조작 방지)
   - `app_config`에 남아 있던 옛 키 삭제

### 3단계 — Edge Function 배포 (대시보드에서 클릭만으로 가능, 설치 불필요)

**③-1. 함수 만들기**
1. Supabase 대시보드 → EngGo 프로젝트 → 왼쪽 메뉴 **⚡ Edge Functions**
2. **Deploy a new function → Via Editor** 클릭
3. 함수 이름: `gemini-proxy` (철자 정확히!)
4. 편집기의 예시 코드를 전부 지우고
   `supabase/functions/gemini-proxy/index.ts` 파일 내용 전체를 붙여넣기
5. **Deploy function** 클릭

**③-2. 키 등록 (Secret)**
1. Edge Functions 화면의 **Secrets** 탭 클릭
2. Add new secret:
   - Key: `GEMINI_API_KEY` (정확히 이 이름)
   - Value: 새로 발급받은 Gemini 키
3. Save

> 참고: Supabase CLI(터미널 명령어)로도 배포할 수 있지만, 함수가 하나뿐이므로
> 대시보드 방식이 가장 간단합니다. 문서에 나오는 Docker는 로컬 개발용이라 필요 없습니다.

### 4단계 — 배포 & 확인
1. 수정된 `index.html`, `service-worker.js`를 GitHub에 다시 올리기
2. 앱에서 로그인 → 영작 교정 한 번 실행 → 정상 응답 확인
3. Supabase 대시보드 → Table Editor → `ai_usage`에 기록이 쌓이는지 확인

### 5단계 — GitHub 저장소 정리 (중요)
키가 한 번이라도 커밋됐다면 **커밋 기록에 영원히 남습니다.**
- 가장 간단한 방법: 저장소를 삭제하고 새로 만들어 현재 파일만 올리기
- 기존 키는 이미 폐기했으므로(1단계) 급하지는 않지만, 새 경고를 막으려면 정리 권장

## 무료 한도 참고 (Gemini 무료 등급)

| 모델 | 분당 요청 | 일일 요청 |
|---|---|---|
| gemini-2.5-flash-lite (현재 사용) | 15 | 1,000 |
| gemini-2.5-flash | 10 | 250 |

- 앱 자체 제한: 호출 간격 4초 + 1인 하루 40회 (클라이언트) / 40회 (서버)
- 학생 25명이 하루 40회씩 써도 1,000회 한도 안에 들어옵니다
- 한도를 바꾸려면: `index.html`의 `AI_DAILY_MAX`, `supabase/functions/gemini-proxy/index.ts`의 `DAILY_LIMIT_PER_USER`

## 개인 키 폴백

프록시 연결이 안 될 때(미배포 등)는 설정 화면에서 개인 Gemini 키를 입력하면
그 기기에서만 직접 호출합니다. 키는 localStorage에만 저장되고 서버로 전송되지 않습니다.
